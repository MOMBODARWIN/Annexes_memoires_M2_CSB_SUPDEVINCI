1. Synthèse exécutive

L'analyse SharpHound du dump SharpHound v4.3.1 (collecte du 10 juillet 2026) confirme et amplifie le constat critique établi par PingCastle (Domain Risk Level 100/100). L'environnement Active Directory de Barkene présente 8 findings exploitables, dont 4 critiques, permettant à un attaquant disposant d'un simple compte utilisateur de compromettre l'intégralité du domaine FINEGIL.local en moins de 30 minutes.

Cibles :

  • Domaine : FINEGIL.local
  • Contrôleurs de domaine : SRV-01-ADY-B-01 (172.21.204.10) et SRV-21-ADY-B-01 (172.20.103.11)
  • Niveau fonctionnel : Windows Server 2016, schéma 2025

Engagement : Signé le 19/06/2026 par Antoine RODRIGUEZ (RSSI) et Alexandre LITAUDON (DSI).

Scénario de compromission simulée : Vol d'une VM Windows 11 jointe au domaine avec le compte ADM.MOMBO (membre du groupe Administrateurs du domaine), SentinelOne désactivé sur le dossier C:\Outils\.

3. Findings détaillés

BH-01 — Compte ADMINISTRATEUR Kerberoastable CRITIQUE — CVSS 9.8

Technique MITRE : T1558.003 — Steal or Forge Kerberos Tickets: Kerberoasting

Description : Le compte ADMINISTRATEUR@FINEGIL.LOCAL (compte builtin Administrator du domaine) possède un Service Principal Name enregistré sur le service MSSQL :

SPN: MSSQLSvc/ServeurW2012R2E.FINEGIL.local:56634

Ce SPN rend le compte Kerberoastable. Tout utilisateur authentifié du domaine peut demander un ticket TGS pour ce SPN, exporter le hash et tenter de le cracker hors-ligne (Hashcat mode 13100).

Exploitation :

# Étape 1 : Demander le TGS
impacket-GetUserSPNs FINEGIL.local/ANY_USER -request -dc-ip 172.21.204.10

# Étape 2 : Cracker le hash
hashcat -m 13100 tgs_administrateur.txt rockyou.txt --force

# Étape 3 : Pass-the-Hash vers le DC
impacket-psexec -hashes :NTLM_HASH ADMINISTRATEUR@172.21.204.10

Impact : Si le mot de passe est cracké → compromission totale du domaine (compte Administrator = DCSync natif, Golden Ticket).

Recommandation :

  • 🔴 P0 : Reset immédiat du mot de passe ADMINISTRATEUR (32+ caractères)
  • 🔴 P0 : Retrait du SPN MSSQL — créer un compte de service dédié (gMSA)
  • 🔴 P0 : Désactiver le compte Administrator au profit d'un compte admin nominatif (Tier 0)
BH-02 — WriteDacl sur domaine pour Exchange Organization Administrators CRITIQUE — CVSS 9.9

Techniques MITRE : T1003.006 (DCSync) + T1222.001 (File/Directory Permissions Modification)

Description : Le groupe Exchange Organization Administrators (RID 3620) dispose d'un WriteDacl sur l'objet racine du domaine CN=FINEGIL,DC=FINEGIL,DC=LOCAL. Ce droit permet à ses membres de modifier la DACL du domaine et donc d'accorder le privilège DCSync à n'importe quel compte.

ACL critiques sur l'objet domaine :

# Étape 1 : Ajouter un utilisateur compromis au groupe Exchange Organization Administrators
net rpc group addmem "Exchange Organization Administrators" USER -S DC_IP

# Étape 2 : DCSync
impacket-secretsdump FINEGIL/USER:PASS@172.21.204.10 -just-dc-user KRBTGT

# Étape 3 : Golden Ticket → compromission totale
# Avec le hash KRBTGT récupéré, générer un Golden Ticket valide 10 ans

Recommandation :

  • 🔴 P0 : Nettoyer les permissions Exchange héritées (script Microsoft Exchange Best Practices)
  • 🔴 P0 : Supprimer le groupe Exchange Organization Administrators ou le vider
  • 🔴 P0 : Vérifier qu'aucun DC n'est membre de ce groupe
BH-03 — 53 comptes admincount=true dont 31 fantômes ÉLEVÉ — CVSS 8.5

Technique MITRE : T1078.002 — Valid Accounts: Domain Accounts

Description : Le mécanisme AdminSDHolder propage un Security Descriptor restrictif aux membres de groupes privilégiés toutes les 60 minutes. Lorsqu'un compte est retiré d'un groupe privilégié, son attribut admincount reste à 1 sans que la SD soit nettoyée. Résultat : 31 comptes « fantômes » héritent de permissions administratives résiduelles.

Comptes critiques identifiés :

  • AVAMARBACKUPUSER — Compte backup Dell, probablement inutilisé
  • SOPHOS — Compte antivirus legacy
  • U_SYS_NINJA, U_SYS_FORTIGATEFW — Comptes d'infrastructure
  • ADMINLFT — Compte admin legacy
  • 31 comptes ex-administrateurs dont la SD AdminSDHolder n'a jamais été révoquée

Recommandation :

  • 🟠 P1 : Audit complet des 53 comptes admincount=true
  • 🟠 P1 : Nettoyage manuel de l'attribut admincount pour les 31 fantômes
  • 🟠 P1 : Mise en place d'un script de nettoyage périodique AdminSDHolder
BH-04 — 12 machines sur OS non supporté (dont Windows XP) CRITIQUE — CVSS 9.0

Techniques MITRE : T1210 (Exploitation of Remote Services), T1003 (OS Credential Dumping)

Description : 12 machines du parc utilisent des systèmes d'exploitation en fin de vie (EOL) :

  • SRV-APP2 : Windows XP SP3 — EOL depuis avril 2014 (plus de 12 ans)
  • 5 serveurs Windows Server 2008 R2 — EOL depuis janvier 2020
  • 1 poste Windows 7 SP1 — EOL depuis janvier 2020
  • 5 serveurs Windows Server 2012/R2 — EOL depuis octobre 2023

Exploits exploitables :

  • Windows XP : MS08-067 (Conficker), MS17-010 (EternalBlue) — pas de Credential Guard
  • Windows 7 / 2008 R2 : EternalBlue (MS17-010), BlueKeep (CVE-2019-0708)
  • Windows Server 2012 R2 : Zerologon (CVE-2020-1472), PrintNightmare (CVE-2021-1675)

Recommandation :

  • 🔴 P0 : Déconnexion immédiate de SRV-APP2 (Windows XP) du réseau
  • 🟠 P1 : Plan de migration prioritaire pour les 11 autres machines EOL
  • 🟠 P1 : Segmentation réseau des machines non migrables
BH-05 — LAPS absent sur 477 machines (0 %) ÉLEVÉ — CVSS 8.0

Technique MITRE : T1078.001 — Valid Accounts: Default Accounts

Description : Aucune des 477 machines du parc n'a LAPS (Local Administrator Password Solution) installé. Le mot de passe du compte Administrator local est donc identique sur toutes les machines ou suit un pattern prévisible.

Impact : Si un attaquant compromet une machine et extrait le hash NTLM du compte Administrator local, il peut le réutiliser sur les 476 autres machines (lateral movement instantané).

Recommandation :

  • 🟠 P1 : Déploiement LAPS sur l'ensemble du parc (477 machines)
  • 🟠 P1 : GPO de restriction du compte Administrator local
  • 🟡 P2 : Rotation forcée des mots de passe locaux post-déploiement LAPS
BH-06 — Unconstrained Delegation sur les 2 DCs MOYEN — CVSS 6.5

Technique MITRE : T1558.001 — Steal or Forge Kerberos Tickets: Golden Ticket

Description : Les deux contrôleurs de domaine ont la délégation non contrainte (TrustedForDelegation) activée :

  • SRV-21-ADY-B-01.FINEGIL.LOCAL
  • SRV-01-ADY-B-01.FINEGIL.LOCAL

Vecteur d'attaque : Coercition d'authentification (PrinterBug, PetitPotam, Coercer) vers un DC → extraction d'un TGT d'admin → Pass-the-Ticket.

Recommandation :

  • 🟠 P1 : Désactiver l'Unconstrained Delegation sur les deux DCs
  • 🟠 P1 : Activer le flag « Account is sensitive and cannot be delegated » pour tous les comptes privilégiés
BH-07 — 431 comptes avec mot de passe n'expirant jamais (38 %) MOYEN — CVSS 5.5

Technique MITRE : T1078.002 — Valid Accounts: Domain Accounts

Description : 431 comptes sur 1 128 (38 %) ont l'attribut pwdneverexpires activé. Ces comptes ne sont jamais soumis à la politique de rotation des mots de passe, augmentant la fenêtre d'exploitation en cas de vol de credentials.

Recommandation :

  • 🟡 P2 : Audit et justification pour chaque compte pwdneverexpires
  • 🟡 P2 : Migration vers des gMSA pour les comptes de service
  • 🟡 P2 : Application d'une politique de rotation stricte
BH-08 — 5 comptes avec mot de passe non requis MOYEN — CVSS 5.0

Technique MITRE : T1078.001 — Valid Accounts: Default Accounts

Description : 5 comptes ont l'attribut passwordnotreqd activé, ce qui signifie qu'ils peuvent légalement avoir un mot de passe vide. SharpHound en a détecté 5 alors que PingCastle n'en avait trouvé que 3.

Recommandation :

  • 🟡 P2 : Identification et correction des 5 comptes concernés
  • 🟡 P2 : Mise en place d'une GPO interdisant les mots de passe vides

4. Chemins d'attaque concrets

Contexte

Scénario de compromission simulée : vol d'une VM Windows 11 sur laquelle l'auditeur est connecté en tant que ADM.MOMBO (compte ayant joint la machine au domaine, admincount=true). SentinelOne est désactivé sur le dossier C:\Outils\.

Chemin 1 : Vol VM → Mimikatz → Lateral Move (LAPS absent)

1
Vol de la VM Windows 11 jointe au domaine FINEGIL.local
2
Extraction des credentials ADM.MOMBO via Mimikatz : sekurlsa::logonpasswords
3
Récupération du hash NTLM (compte admincount=true)
4
LAPS absent → le compte Administrator local a un mot de passe identique ou prévisible sur les 477 machines
5
Scan SMB massif : réutilisation du hash local Administrator sur les 477 machines
6
Pivot vers n'importe quelle machine du parc ✅

Chemin 2 : admincount=true → Exchange WriteDacl → DCSync

1
Compromission de n'importe quel des 53 comptes admincount=true
2
Le groupe Exchange Organization Administrators (RID 3620) a WriteDacl sur la racine du domaine
3
Ajout du compte compromis au groupe Exchange Org Admins
4
DCSync : secretsdump.py FINEGIL/USER:PASS@172.21.204.10
5
Extraction du hash KRBTGT → Golden Ticket
6
Domain Admin total et persistant ✅

Chemin 3 : Kerberoasting ADMINISTRATEUR

1
Authentification avec n'importe quel compte du domaine
2
Demande de ticket TGS pour le SPN MSSQLSvc/ServeurW2012R2E.FINEGIL.local
3
Crack du hash hors-ligne (Hashcat mode 13100)
4
Si cracké → Pass-the-Hash ADMINISTRATEUR vers le DC
5
ntdsutil → extraction NTDS.dit
6
Compromission totale du domaine ✅

Chemin 4 : Attaque serveur Windows XP (SRV-APP2)

1
Compromission initiale (peu importe le vecteur)
2
Lateral move vers SRV-APP2 (Windows XP SP3)
3
Exploitation MS08-067 (Conficker) ou MS17-010 (EternalBlue)
4
Aucune protection : pas de Credential Guard, pas de patchs depuis 2014
5
Mimikatz direct → extraction de tous les credentials en cache
6
Pivot vers le DC si un admin s'est connecté ✅
